Javascript objects are key-value pairs. Each item contained in the object has to have a reference (the key). The keys in javascript objects are referred to as 'properties', which can hold (value) of a string, number, array, function, or another object.

Let's create an object for our Pacman:

var pacman = {};

YOU!!! as the developer, will determine what properties/attributes each object needs and will have. For instance, to keep track of our character let's use x & y coordinates to locate pacman on the gameboard. Take a few minutes to think about what other properties that you could add.

The goal with questions like this is to get you to start thinking like a developer. What would you want to put into your game? 

  • What can a player have? (properties / attributes). 
  • What can a player do? (methods or functions). 

Let's go ahead an populate the object with the x/y coordinate properties:

pacman = {
  x: 5,   // note the comma between the object's properties
  y: 5
}

To change an object's property

pacman.x = 6;

What other properties could your player have? 

  • number of lives
  • score (especially if it is a multiplayer game) 

What other objects could be used to create this game? Remember, as the developer YOU will be determining what objects and properties those objects will have to complete the task you are currently facing.
